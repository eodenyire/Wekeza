# Wekeza Mobile Web (PWA)

A **Progressive Web App (PWA)** for Wekeza Bank's mobile banking service. Built with React + TypeScript + Vite + Tailwind CSS.

This is the **scalable mobile web version** that works in any mobile browser — no app store installation required. Users can also "Add to Home Screen" for a full native-app experience.

## Features

- 📱 **Mobile-first UI** — optimized for small screens with iOS safe area support
- ⚡ **PWA** — installable, works offline, service worker caching
- 🔐 **Authentication** — JWT login with session persistence
- 💳 **Account Management** — view all accounts and balances
- 💸 **Fund Transfers** — bank-to-bank transfers via v1-Core
- 📊 **Transaction History** — paginated statement per account
- 📱 **M-Pesa Integration** — STK push, send to mobile, airtime top-up
- 🏦 **Loans** — view active loans with repayment progress
- 👤 **Profile** — user info and settings

## Getting Started

### Prerequisites

- Node.js >= 18
- npm >= 9

### Install dependencies

```bash
npm install
```

### Configure environment

```bash
cp .env.example .env
# Edit .env and set your API base URL
```

### Run development server

```bash
npm run dev
# Opens at http://localhost:3000
```

The development server proxies all `/api` requests to `http://localhost:5000` (the v1-Core API).

### Build for production

```bash
npm run build
# Output in dist/
```

### Preview production build

```bash
npm run preview
```

## Integration with v1-Core

All API calls are made to the Wekeza v1-Core backend. Configure `VITE_API_BASE_URL` in `.env`:

| Environment | URL |
|-------------|-----|
| Development | `http://localhost:5000/api` |
| Production  | `https://api.wekeza.com/api` |

The app uses the following v1-Core endpoints:
- `POST /authentication/login`
- `GET /accounts/user/accounts`
- `GET /accounts/{accountNumber}/balance`
- `GET /transactions/statement/{accountNumber}`
- `POST /transactions/transfer`
- `GET /loans/user/loans`
- `POST /digitalchannels/mpesa/stk-push`
- `POST /digitalchannels/send`
- `POST /digitalchannels/airtime`
- `POST /payments/bill`

## PWA Configuration

The PWA manifest is generated by `vite-plugin-pwa` and includes:
- App name, icons, theme color
- Service worker with network-first caching for API calls
- Offline fallback for static assets
- Standalone display mode (full screen, no browser chrome)
- iOS splash screen support
