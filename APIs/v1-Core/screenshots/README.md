# v1-Core Screenshots

This folder contains screenshots of every major portal and screen in the
**Wekeza v1-Core Core Banking API** system.

Screenshots are generated by the offline screenshot generator located at
[`tests/e2e/take_screenshots.py`](../tests/e2e/take_screenshots.py).

## Folder Structure

| Folder | Description |
|--------|-------------|
| `01_login/` | Login page and authentication error state |
| `02_dashboard/` | System-wide admin dashboard with KPIs and alerts |
| `03_accounts/` | Account registry list and individual account detail |
| `04_transactions/` | Full transaction ledger with filters |
| `05_teller_portal/` | Teller counter – deposit and withdrawal forms |
| `06_loans/` | Loan register and new loan application form |
| `07_customer_portal/` | Customer self-service portal view |
| `08_branch_manager/` | Branch manager overview and teller summary |
| `09_admin_panel/` | User management and system configuration |
| `10_reports/` | Financial reports – income statement and balance sheet |
| `11_compliance/` | AML alerts and compliance dashboard |
| `12_payments/` | M-Pesa and RTGS/EFT payments portal |
| `13_api_docs/` | Swagger / OAS 3.0 API documentation |

## Regenerating Screenshots

To regenerate all screenshots, run:

```bash
python3 APIs/v1-Core/tests/e2e/take_screenshots.py
```

Requirements: `pip install playwright && playwright install chromium`

The script runs in **offline mode** (no live server required) – it renders
self-contained HTML mock pages that faithfully replicate the v1-Core admin/portal
web UI.  This is ideal for CI, documentation and code-review workflows.

---

*Each screenshot uses a 1440 × 900 desktop viewport to match the v1-Core admin panel layout.*
